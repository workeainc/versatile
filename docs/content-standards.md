# Content Standards

These standards keep the site consistent as projects, local landing pages, and articles grow.

## Projects

- Minimum 6 gallery images when real photography is available.
- 300-600 words of project story.
- Include location, year, area, completion date, client type, and status.
- Include client goals, constraints, design process, material choices, lighting strategy, and results.
- Include one client quote only when permission is confirmed.
- List services provided and primary materials.
- Use real project photography whenever possible.
- Avoid inflated metrics or unverified awards.

## Blog Articles

- Target 1,200-2,000 words for major SEO articles.
- Use one primary keyword and several natural supporting terms.
- Include internal links to related projects, services, local pages, and other articles.
- Add FAQ content when the topic has clear client questions.
- Use original photos where possible; otherwise replace stock images before launch.
- Add clear CTA language near the end when relevant.
- Keep advice specific to Bangladesh when the search topic is local.
- Add or update `seoTitle` and `seoDescription` in frontmatter.

## Local SEO Pages

- Do not publish thin location pages.
- Each page should include a localized introduction, relevant services, featured projects, FAQs, and a contact CTA.
- Mention the location naturally, not mechanically.
- Include structured data and sitemap coverage.
- Use real local projects as soon as they are available.

## Trust Signals

- Use only verified project counts, years of experience, awards, certifications, and client logos.
- Omit testimonials until permission is clear.
- Prefer specific proof over generic claims.
- Replace placeholders before launch wherever the client can provide real content.

## Image Standards

- Prefer real project photos over Unsplash before launch.
- Keep hero images sharp at desktop sizes.
- Use descriptive alt text.
- Avoid stretched or heavily cropped images where details matter.
- Store final project assets in a predictable media structure when real content is ready.
