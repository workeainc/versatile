# Project Image Workflow

Use this workflow when replacing placeholder project content with real photography.

## Folder Structure

Create one folder per project:

```text
src/media/projects/project-slug/
```

Example:

```text
src/media/projects/courtyard-house/
```

Inside that folder, place the original highest-quality images.

Do not rename, crop, compress, or resize images before adding them. Original files give us the best source for web exports, crops, Open Graph images, and future reuse.

## Recommended Source Files

Add whatever is available:

- Final project photography
- Detail shots
- Before photos
- After photos
- Floor plans if permitted
- Construction progress photos
- Material palette images
- Lighting/detail photos
- Walkthrough video links in a text file if available

Optional text file:

```text
src/media/projects/project-slug/project-info.txt
```

## What Codex Will Produce

For each project, Codex will:

- Choose the strongest hero image.
- Select gallery order.
- Separate wide shots, detail shots, before/after images, and process images.
- Crop images to consistent aspect ratios.
- Resize images for web performance.
- Export optimized formats.
- Generate social/Open Graph image versions.
- Produce SEO-friendly filenames.
- Write descriptive alt text and captions.
- Update project data and page content.
- Update structured data image references.
- Add internal links to relevant services, articles, and local pages.

## Image SEO Output Format

Each image should receive:

```text
Source:
IMG_4921.JPG

Export:
dhaka-courtyard-house-living-room.webp

Alt:
Open-plan living room with natural oak finishes in Courtyard House, Dhaka

Caption:
The living area was planned to improve daylight and create visual continuity between family spaces.

Usage:
Hero image / gallery image / detail image / Open Graph image
```

## Export Naming Rules

Use lowercase, descriptive filenames:

```text
location-project-room-or-detail.webp
```

Examples:

```text
dhaka-courtyard-house-living-room.webp
gulshan-courtyard-house-dining-detail.webp
banani-quiet-office-meeting-room.webp
dhanmondi-ember-dining-room-banquette.webp
```

## Web Export Targets

Typical outputs per project:

- Hero image
- Cover image
- Gallery images
- Thumbnail image
- Open Graph image
- Before/after pairs if available

The exact sizes should be selected after reviewing the source photos and page usage.

## Important Rules

- Do not fake before/after images.
- Do not use client names without permission.
- Do not use testimonials without permission.
- Do not publish floor plans if the client considers them private.
- Do not invent metrics, awards, suppliers, or timelines.
- Replace stock images only when real project images are strong enough.
